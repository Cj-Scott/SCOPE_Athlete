Workflow A / Parent onboarding

Can a student-athlete family understand the first path?

Use this script to test whether a family can orient themselves, trust the data boundary, and understand what to do first.

20 minutes Scenario: first family review Settings + Profile + Overview

Scenario

A parent opens the tool for the first time.

The family wants to understand what SCOPE Athlete does, whether synthetic demo records are safe to explore, and whether the profile/dashboard explain the athlete's current planning picture.

Experience goal

The app should feel calm, guided, and private.

A reviewer should know where they are, what data is local, why demo data exists, and which workspace to open next without needing a walkthrough narrator.

Feature touchpoints

  • Install and first launch
  • Settings and Load demo data
  • Global player selector and profile image
  • Profile workspace summary
  • Overview dashboard and backup language

Walkthrough script

Follow these steps in order.

  1. 01
    Open the alpha build as a first-time parent.

    Before clicking around, note whether the app makes its purpose and SCOPE acronym understandable.

  2. 02
    Load demo data in Settings.

    Confirm the copy makes it obvious that records are synthetic, local, and replace current local records.

  3. 03
    Open Player Profile.

    Review the player photo, graduation year, position, academic fields, soccer fields, and whether the profile feels like a useful family snapshot.

  4. 04
    Switch profiles.

    Confirm the selector feels obvious and each name/photo change is visible across the app.

  5. 05
    Open Overview.

    Judge whether the dashboard summarizes status and next actions instead of asking you to edit records there.

  6. 06
    Return to Settings.

    Check backup, logging, version, updates, and privacy language for parent-level clarity.

  7. 07
    Name the next action.

    Without using this guide, decide what a family should do next and whether the app made that obvious.

Quick check

  • Clear start

    Could a parent begin without help?

  • Safe data

    Was the demo-data boundary easy to trust?

  • Useful summary

    Did Overview explain the school-fit picture?

  • Journey confidence

    Did the experience feel like a coherent first session?

Feedback prompts

Answer these after the walkthrough.

  • What felt clear?

    Name the first screen or label that helped you understand the app.

  • What felt unsafe or unclear?

    Call out any data, backup, privacy, or demo-data language that caused hesitation.

  • Could a parent tell what to do next?

    Describe the next action you expected after reviewing the profile and overview.

  • Where did the journey break?

    Note the first moment you felt lost, over-informed, or unsure whether to trust the screen.

  • What should be simplified?

    Point to any setting, status, or card that felt too technical for alpha reviewers.

If you get stuck

Click-by-click fallback path.

  1. Open Settings.

    Use the left navigation menu and select Settings.

  2. Load the demo.

    Find the backup/demo area and select Load demo data. Confirm the warning if prompted.

  3. Open Profile.

    Use the left navigation menu and select Profile.

  4. Switch players.

    Use the player selector near the top-left app header. Choose a second demo player and confirm the profile image/name change.

  5. Open Overview.

    Select Overview in the left navigation and scan the summary cards before editing anything.

  6. Return to Settings.

    Select Settings, then review backup, diagnostics, logging, updates, and version cards.

In feedback, note every point where you needed these fallback instructions instead of finding the path naturally.
