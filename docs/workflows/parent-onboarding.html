Workflow A / Parent onboarding

Can a student-athlete family understand the first path?

Use this script to test whether profile, overview, backup, and privacy language feel safe and clear.

20 minutes Settings + Profile + Overview Synthetic demo data only

Walkthrough script

Follow these steps in order.

  1. 01
    Open the alpha build.

    Start from a clean launch so the first impression is fresh.

  2. 02
    Load demo data in Settings.

    Confirm the copy makes it obvious that records are synthetic and local.

  3. 03
    Open Player Profile.

    Review the player photo, graduation year, position, academic fields, and soccer fields.

  4. 04
    Switch profiles.

    Confirm the selector feels obvious and the image/name update across the app.

  5. 05
    Open Overview.

    Judge whether the dashboard reads as a summary instead of an editing workspace.

  6. 06
    Return to Settings.

    Check backup, logging, version, and privacy language for parent-level clarity.

Quick check

  • Clear start

    Could a parent begin without help?

  • Safe data

    Was the demo-data boundary easy to trust?

  • Useful summary

    Did Overview explain the school-fit picture?

Feedback prompts

Answer these after the walkthrough.

  • What felt clear?

    Name the first screen or label that helped you understand the app.

  • What felt unsafe or unclear?

    Call out any data, backup, privacy, or demo-data language that caused hesitation.

  • Could a parent tell what to do next?

    Describe the next action you expected after reviewing the profile and overview.

  • What should be simplified?

    Point to any setting, status, or card that felt too technical for alpha reviewers.
