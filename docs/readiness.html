Alpha readiness

What gets checked before a reviewer wave.

This is a simple public-facing version of the go/no-go checklist. It explains the review gate without exposing private release details or backend operations.

Go / no-go gate

The alpha should only be shared when these checks pass or are explicitly waived.

For the current pause window, installer downloads are paused until August 10, 2026. Videos, walkthroughs, FAQ, and feedback are still available for early review.

Paused

Installer availability

EXE and MSI download buttons are paused until the next review window opens on August 10, 2026.

Ready

Reviewer orientation

The landing page, Alpha Hub, workflow pages, FAQ, and demo videos should explain what to review and where to start.

Ready

Synthetic demo boundary

Reviewer materials should tell people to use synthetic demo data only and avoid entering private athlete or family information.

Check

App update check

Before sharing an installer wave, verify the installed app can check for the expected signed app update status.

Check

Data-pack update check

Verify school catalog, rules, and evidence data-pack checks show clear status and do not break the existing local experience.

Check

Demo data load/reset

Confirm a clean install can load demo data and reset the alpha experience without requiring real user records.

Check

Backup and diagnostics

Confirm backup export and diagnostics export show the correct status, location guidance, and active theme styling.

Check

Clean uninstall

Confirm Windows uninstall removes the app cleanly and that any local data retention behavior is understood before review.

Ready

Feedback path

The Google Form link should open, and feedback should be manually reviewed before it becomes product work.

Run record

Suggested pre-share note.

Before inviting reviewers, record the date, app version, data-pack versions, reviewer wave, pass/fail result, and any waived items in a private working note.
